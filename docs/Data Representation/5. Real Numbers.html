  1. Rational Number Approximation - any real number can be approximated by a rational number, so we already have established a way to represent real numbers to a certain accuracy

  2. Floating-Point Representation - the base-two equivalent of scientific notation where a real number x is represented by σb×2e with b and e being integers

    • mantissa - the number b with bit-length k specifies a sequence of bits
    • exponent - the number e with bit-length l specifies the location of where to put the floating point in the bit sequence provides by b
    • sign - σ∈{±1} which gets mapped to a 1 bit for a negative number and to 0 for a positive number

  3. Cantor's Theorem - there is no injection RtS:R→{0,1}∗, meaning that it is impossible to exactly represent real numbers as strings

    Proof:

    To prove the theorem we will construct an infinite set X for which it is easier to prove that there is no one-to-one map from it to {0,1}∗. Then, we will prove that there is a one-to-one map G:X→R.

    This is essentially a proof by contradiction. Suppose there is a function F:R→{0,1}∗. Then, the composite function F(G(x)) would be a one-to-one map from X to {0,1}∗ which is a contradiction because we will prove that there is no such map.

    Now, let {0,1}∞ represent the set of all functions which take a natural number as an input and produce either a 0 or a 1. That is,

    {0,1}∞:={f∣f:N0​→{0,1}}

    Now we prove a lemma stating that there is no injection "Functions to Strings" ( FtS:{0,1}∞→{0,1}∗ ) which is equivalent to showing that there is no surjection "Strings to Functions" ( StF:{0,1}∗→{0,1}∞ ).

    Assume there is such a function StF:{0,1}∗→{0,1}∞. Consider the lexicographic ordering of binary strings ("",0,1,00,11,...) and let xn​(n∈N0​) denote the n-th member of this tuple. We define the following function dˉ∈{0,1}∞:

    dˉ(n)=1−StF(xn​)(n) This is a lot to unpack. Essentially StF(xn​) returns a function corresponding to the string xn​, which we will call gxn​​∈{0,1}∞, and then this returned function is called with n. The above definition can be rewritten as:

    dˉ(n)=1−gxn​​(n),gxn​​(n)=StF(xn​)

    Since g(n) returns either a 0 or a 1, dˉ(n) can be thought of as the negation of gxn​​(n). All this can be represented in table form:

    This table is essentially a definition of StF. Each row corresponds to a function gi​=StF(i),i∈{x0​,x1​,x2​,...} and each column corresponds to a number n∈N0​ which is passed to the functions gi​. Each cell is then equal to gi​(n)=StF(xn​)(n).

    By definition, dˉ(n)=1−StF(xn​)(n), which means that dˉ operates on the diagonal values of the table. Additionally, since dˉ(n) is the negation of gxn​​(n), then dˉ(n) must differ from every possible function gxn​​ for at least one input. But this means that we have just managed to construct a function which is not in the table and therefore is not covered by StF. Therefore, there is no function StF which is onto {0,1}∞ because dˉ:N0​→{0,1}∈{0,1}∞ but we just showed that dˉ is not in the image of StF. This means that we have shown that there is no one-to-one map FtS:{0,1}∞→{0,1}∗.
