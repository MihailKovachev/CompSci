  3. Representation as a String - a representation scheme for any set O is a pair of functions (E,D) where E:O→{0,1}∗ is a total one-to-one function, D:{0,1}∗→p​O is a possibly partial function and such that D(E(o))=o,∀o∈O.

    a) encoding function - E

    • every injective encoding function has a corresponding decoding function - let E:O→{0,1}∗ be one-to-one. Then there exists a function D:{0,1}∗→O such that D(E(o))=o,∀o∈O.

    Proof:

    Let o0​ be an arbitrary element of O. Since E is one-to-one, then for every x∈{0,1}∗ there exist either zero or a single o∈O such that E(o)=x. The decoding function can then be defined for those two cases follows:

    D(x)={o0​, when there is no o∈O such that E(o)=xo, when there is an o∈O such that E(o)=x​

    This satisfies the condition that D is onto O.

    b) decoding function - D

    • surjective - D(E(o))=o,∀o∈O means that D must cover the entirety of O because E is a total one-to-one function
    • can technically be partial but is always total for any valid representation scheme due to the proof showing that every one-to-one encoding function has a decoding function

    c) finite representations - if O is finite, then every object in it can be represented by a string of length at most n, which is in the set {0,1}≤n:={x∈{0,1}∗:∣x∣≤n}

    • encoding function - this needs to be a one-to-one map from O to {0,1}≤n, which is only possible if ∣O∣≤∣{0,1}≤n∣=2n+1−1 due to the pigeonhole principle

  3. Prefix-free Encoding - a function E:O→{0,1}∗ which maps a non-empty set O to {0,1}∗ is prefix-free if E(o)="",∀o∈O and there is no pair of objects o,o′∈O such that E(o) is a prefix of E(o′)

    a) tuple encoding - if there is a prefix-free encoding E:O→{0,1}∗, then it is always possible to define a one-to-one encoding (that, however, is not itself prefix-free) which maps a finite tuple of objects in O to a string

    Eˉ:O∗→{0,1}∗

    Eˉ(o0​,o1​,...,ok−1​):=E(o0​)E(o1​)⋯E(ok−1​)

    Proof:

    Suppose there are two distinct tuples (o0​,o1​,...,ok−1​) and (o0′​,o1′​,...,ok′−1′​) such that

    Eˉ(o0​,o1​,...,ok−1​)=Eˉ(o0′​,o1′​,...,ok′−1′​)

    We said the tuples were distinct, so let i denote the smallest index for which oi​=oi′​.

    If there is no such index, then one of the tuples must be larger (we assume without loss of generality that k′>k in this case).

    Consider the case where i<k. Then the following is true:

    Eˉ(o0​,o1​,...,ok−1​)=E(o0​)⋯E(oi−1​)E(oi​)E(oi+1​)⋯E(ok−1​)

    Eˉ(o0′​,o1′​,...,ok′−1′​)=E(o0​)⋯E(oi−1​)E(oi′​)E(oi+1′​)⋯E(ok′−1′​)

    The two representations share a common prefix x and so can be written as follows:

    Eˉ(o0​,o1​,...,ok−1​)=xE(oi​)E(oi+1​)⋯E(ok−1​)

    Eˉ(o0′​,o1′​,...,ok′−1′​)=xE(oi′​)E(oi+1′​)⋯E(ok′−1′​)

    Let y denote the rest of the representation string after removing the prefix. Note that y is identical for both representations since we assumed they were identical. The string y can be written in the following two ways:

    {y=E(oi​)s,s∈{0,1}∗y=E(oi′​)s′,s′∈{0,1}∗​⟹E(oi​)s=E(oi′​)s′

    This implies, however, that either E(oi​) or E(oi​)′ has to be a prefix of the other, thus contradicting the prefix-freeness of E.

    In the case when i=k, we have that

    E(o0​)E(o1​)⋯E(ok−1​)=E(o0​)E(o1​)⋯E(ok−1​)E(ok′​)⋯E(ok′−1′​)

    This means that E(ok′​)⋯E(ok′−1′​) must be the empty string, thus contradicting once again the prefix-freeness of E.

    Both cases led to a contradiction, so the only possible conclusion is that there are no two distinct tuples for which

    Eˉ(o0​,o1​,...,ok−1​)=Eˉ(o0′​,o1′​,...,ok′−1′​)

  1. String Tuples as Binary Strings - if an object of type T can be represented as a string, then so can an n-tuple of objects of type T

    a) mapping delimiter characters - the strings forming a tuple need to be separated with a delimiter character which augments the alphabet set

    Example: There exists an injection which maps pairs of binary strings x,y∈{0,1}∗ to a single string z∈{0,1,∥}∗ which represents a tuple.

    Consider the strings 10 and 01101. They can be represented as a tuple by 10∥01101 where ∥ is the chosen delimiter.

    Now this tuple needs to itself be represented as a binary string. This can be done by mapping {0,1,∥} to {00,11,01}⊂{0,1}∗. That way, a length n string with three possible characters gets mapped to a string of length 2n with two possible characters. The example tuple 01∥01101 will then get mapped to 0011010011110011.

  2. Rational Numbers - any rational number is essentially a tuple a/b with / as the delimiter

    a) as binary strings

    • represent a and b as binary strings using signed magnitude representation
    • represent the rational number as a tuple of binary strings
    • use the string-tuple-to-binary-string mapping to turn the tuple into a single string

    Example: Consider the number −5/8. We represent −5 as 1101 and +8 as 01000. These represent the tuple 1101/01000 and we can map this to a binary string 11110011010011000000

  1. Rational Number Approximation - any real number can be approximated by a rational number, so we already have established a way to represent real numbers to a certain accuracy

  2. Floating-Point Representation - the base-two equivalent of scientific notation where a real number x is represented by σb×2e with b and e being integers

    • mantissa - the number b with bit-length k specifies a sequence of bits
    • exponent - the number e with bit-length l specifies the location of where to put the floating point in the bit sequence provides by b
    • sign - σ∈{±1} which gets mapped to a 1 bit for a negative number and to 0 for a positive number

  3. Cantor's Theorem - there is no injection RtS:R→{0,1}∗, meaning that it is impossible to exactly represent real numbers as strings

    Proof:

    To prove the theorem we will construct an infinite set X for which it is easier to prove that there is no one-to-one map from it to {0,1}∗. Then, we will prove that there is a one-to-one map G:X→R.

    This is essentially a proof by contradiction. Suppose there is a function F:R→{0,1}∗. Then, the composite function F(G(x)) would be a one-to-one map from X to {0,1}∗ which is a contradiction because we will prove that there is no such map.

    Now, let {0,1}∞ represent the set of all functions which take a natural number as an input and produce either a 0 or a 1. That is,

    {0,1}∞:={f∣f:N0​→{0,1}}

    Now we prove a lemma stating that there is no injection "Functions to Strings" ( FtS:{0,1}∞→{0,1}∗ ) which is equivalent to showing that there is no surjection "Strings to Functions" ( StF:{0,1}∗→{0,1}∞ ).

    Assume there is such a function StF:{0,1}∗→{0,1}∞. Consider the lexicographic ordering of binary strings ("",0,1,00,11,...) and let xn​(n∈N0​) denote the n-th member of this tuple. We define the following function dˉ∈{0,1}∞:

    dˉ(n)=1−StF(xn​)(n) This is a lot to unpack. Essentially StF(xn​) returns a function corresponding to the string xn​, which we will call gxn​​∈{0,1}∞, and then this returned function is called with n. The above definition can be rewritten as:

    dˉ(n)=1−gxn​​(n),gxn​​(n)=StF(xn​)

    Since g(n) returns either a 0 or a 1, dˉ(n) can be thought of as the negation of gxn​​(n). All this can be represented in table form:

    This table is essentially a definition of StF. Each row corresponds to a function gi​=StF(i),i∈{x0​,x1​,x2​,...} and each column corresponds to a number n∈N0​ which is passed to the functions gi​. Each cell is then equal to gi​(n)=StF(xn​)(n).

    By definition, dˉ(n)=1−StF(xn​)(n), which means that dˉ operates on the diagonal values of the table. Additionally, since dˉ(n) is the negation of gxn​​(n), then dˉ(n) must differ from every possible function gxn​​ for at least one input. But this means that we have just managed to construct a function which is not in the table and therefore is not covered by StF. Therefore, there is no function StF which is onto {0,1}∞ because dˉ:N0​→{0,1}∈{0,1}∞ but we just showed that dˉ is not in the image of StF. This means that we have shown that there is no one-to-one map FtS:{0,1}∞→{0,1}∗.
